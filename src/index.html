Mandelbrot with wgpu-rs

Tried to implement my own Mandelbrot set explorer with compute shaders, however:

  • WebGPU isn't ready yet
  • Boundary tracing is an inherently serial workload, and GPUs are built for parallel workloads
  • Other sites exist that use fragment shaders for the same purpose

Due to these limitations, I've stopped working on it until we can finally use compute shaders on the web. There's a plethora of ways to optimize a Mandelbrot set, but they're not universally beneficial. Ideally, the CPU should predict which pixels actually need to be processed and hand that off to the GPU. Whether its symmetry, perodicity, or boundary tracing. In reality, a good GPU is so fast that it can render multiple frames per second.

It was still a cool project, and I learned things about complex numbers, iterators and collections in Rust, and NOT mutably borrowing within nested parallel iterators. Oh, I also made an unsafe function that mutates a raw pointer from a Vec and it worked, but it was slower.

MAF calibration CLI

So a buddy of mine has a Mazdaspeed3 and he likes to tinker with it. He got really annoyed with tuning, then test driving to generate some logs, then editing every column and row in his tune file, and repeating this to improve his vehicle's performance.

He wrote this Python program to fit a curve to his log data, make some corrections, and pump out a new CSV file that he can put into his tune. Wicked.

I was like "watch me re-write it in Rust with a GPU compute shader", and then I did.

There was absolutely no good reason to optimize his program further with Rust and wgpu-rs. I just wanted an excuse to use a compute shader.
